kiri-ux

A token-driven, dark-mode-first design system. Vanilla CSS + minimal JS. Rotate the brand hue in the sidebar — every component re-tints coherently because nothing hardcodes a color.

Buttons

✅ Use for: actions the user takes. Primary = the one main action per view.

🚫 Don't: use more than one primary button in a view, or use a button for navigation (use a link).

<button class="btn btn--primary">Primary</button>

Badges

✅ Use for: status, category, or verdict labels inline with content.

🚫 Don't: use as a clickable control (it's a label, not a button).

Neutral Brand True False Misleading Info 
<span class="badge badge--success">True</span>
